# v1.0.5 Release Notes

## GitHub Skills Importer

New `lucidRESUME.GitHub` module that imports skills from your GitHub profile:

- **Languages**: weighted by bytes across all repos (C# 65%, JavaScript 31%, etc.)
- **Topics**: from repo metadata (docker, rag, llm, etc.)
- **README parsing**: lucidRAG DocSummarizer (BERT mode, no LLM needed) extracts technologies from README content
- **Per-project profiles**: each repo gets a structured profile with technologies, skills, time range, evidence strength, and summary
- **Rate limit enforcement**: `EnsureBudget()` before each repo, graceful degradation with partial results
- **PAT auth**: configure `GitHub:PersonalAccessToken` via user secrets for 5000 req/hr + private repo access
- **IT-only skill filtering**: blocks false positives from non-tech taxonomies

### CLI

```bash
lucidresume github-import --username scottgal
```

### GUI

New "GitHub Import" card on the Profile page — enter a username, click Import.

## My Data Page

New page between "My CV" and "Jobs" showing all extracted data with evidence drill-down and corrections:

- **Personal Info card**: all extracted fields (resume + GitHub), editable
- **Skill Ledger card**: category filter, search, tier badges (strong/moderate/weak), expand any skill to see every piece of evidence with source type, company, date range, confidence
- **Experience timeline**: work history from resume
- **Projects card**: GitHub repos with technologies
- **Consistency Issues**: warnings about unsubstantiated skills

### Corrections

- **Dismiss skills or individual evidence** — corrections stored in `UserOverrides`, separate from resume data so re-imports don't lose them
- **Add manual skills** — creates evidence with `EvidenceSource.Manual`
- **Skill renames** — correct misidentified skill names
- **Autosave** — 800ms debounce, same pattern as Profile page

## lucidRAG Integration

Added [lucidRAG](https://github.com/scottgal/lucidrag) as a git submodule for README analysis:

- `IDocumentSummarizer.SummarizeMarkdownAsync()` in BERT mode — extractive summarization without LLM
- Topic extraction, segment analysis, and entity detection from README markdown
- Project descriptions enriched from lucidRAG executive summaries

## UX Testing Improvements

Scroll action now supports multiple modes:

| Syntax | Effect |
|--------|--------|
| `top` / `bottom` | Jump to start/end |
| `up` / `down` | Page up/down (full viewport) |
| `50%` | Scroll to percentage of content |
| `300` | Scroll to absolute pixel position |
| `+200` / `-100` | Relative pixel offset |

## Other Changes

- `SkillLedgerBuilder.MergeEvidence()` — merges GitHub skills into resume skill ledger
- `SkillLedgerBuilder.ApplyOverrides()` — applies user corrections (dismissals, renames, manual additions)
- `SkillCategoriser.CategoriseSkill()` — standalone single-skill categorisation
- `EvidenceSource.GitHubRepository` and `EvidenceSource.Manual` enum values
- NuGet package URL corrected for `Mostlylucid.Avalonia.UITesting`
- CLI now reads user secrets (shared secrets ID with desktop app)
