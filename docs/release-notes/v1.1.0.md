# v1.1.0 Release Notes

## GitHub Skills Importer

Import skills directly from your GitHub profile:

- **Languages**: weighted by bytes across all repos
- **Topics**: from repo metadata
- **README parsing**: [lucidRAG](https://github.com/scottgal/lucidrag) DocSummarizer (BERT mode, no LLM needed) extracts technologies from README content
- **Per-project profiles**: each repo gets a structured profile with technologies, skills, time range, and evidence strength
- **Rate limit enforcement**: graceful degradation with partial results
- **PAT auth**: 5000 req/hr + private repos via user secrets
- CLI: `lucidresume github-import --username scottgal`
- GUI: GitHub Import card on Profile page

## LinkedIn Data Export Import

Drop your LinkedIn ZIP archive and it auto-imports:

- Positions with achievements, technologies
- Skills with endorsement counts
- Education, projects, contact info
- Auto-detected from ZIP contents (no manual format selection)

## Drag-and-Drop File Import

Drop any file on the main window:
- `.pdf`, `.docx`, `.txt` → resume parser
- `.zip` → LinkedIn detector (auto-routes)

## My Data Page

New dashboard between "My CV" and "Jobs" showing all extracted data:

- **Skills Dashboard**: category pie chart (LiveCharts2), top skills radar, strength tier badges
- **Career Timeline**: Gantt-style colored bars per role
- **Skill Ledger**: category filter, search, expand for evidence drill-down (source, company, dates, confidence)
- **Corrections**: dismiss skills/evidence, add manual skills, rename — stored in UserOverrides
- **Experience**: deduplicated across imports with variant storage for user review
- **Projects**: from GitHub import
- **Consistency Issues**: unsubstantiated skill warnings

## Career Planner Page

New "Career" page with gap analysis against a selected job:

- **Quick Wins** (low effort): presentation gaps, weak evidence — just update wording
- **Side Projects** (medium effort): adjacent skills in same Leiden community
- **New Learning** (high effort): true gaps requiring dedicated study
- Fit/coverage/actions-to-good-fit badges

## DOCX Preview via Morph

Cross-platform DOCX-to-image rendering powered by [Morph](https://github.com/SimonCropp/Morph):
- Pure C#, SkiaSharp, no LibreOffice needed
- Preview fallback: Docling → Morph → LibreOffice → no preview

## DOCX + PDF Export

- **DOCX**: professionally formatted via DocumentFormat.OpenXml
- **PDF**: [QuestPDF](https://www.questpdf.com/) with styled headings, experience sections, skill groups
- CLI: `lucidresume export --format docx|pdf --output resume.docx`

## Leiden Community Detection

Upgraded from Louvain to Leiden with refinement phase — produces higher-quality, more stable skill clusters.

## Resume Extraction RRF Fusion

Entities detected by multiple extractors (Pattern + NER + LLM) now get confidence-boosted via reciprocal rank fusion, matching the JD field fuser pattern.

## Temporal Skill Drift

`SkillDriftAnalyser.Compare()` detects skills added, dropped, strengthened, or weakened across resume variants.

## Experience Deduplication

Cross-import deduplication matches by company name similarity + date overlap:
- Merges technologies and unique achievements
- Strips company suffixes (Ltd, Inc, Corp, etc.) for fuzzy matching
- Stores both variants for user review when there's a conflict

## Other Improvements

- LiveCharts2 dashboard visualizations (category pie, radar, Gantt)
- UX testing: scroll supports percentage, pixel, and page offsets
- CI: submodule checkout for lucidRAG
- Release workflow: GitHub release on both tag push and manual dispatch
- Pre-converted resume NER model on HuggingFace (no Python needed)
- User-focused README with download instructions
