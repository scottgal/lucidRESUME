# v1.5.0 Release Notes

## Skill Taxonomy — 19,983 Skills from 1.3M Real Jobs

Preloaded skill centroids mined from Kaggle datasets using DuckDB:
- **9,444 role-specific skills** across 16 job archetypes (Backend Developer, DevOps Engineer, Machine Learning, etc.) from Kaggle `job_skills_updated.xlsx`
- **10,539 cross-industry skills** extracted from 1.3M LinkedIn job postings (`job_skills.csv`) — nursing, accounting, sales, engineering, everything — not just tech
- **1,542 priority-classified skills** (High/Medium/Low) for archetype weighting

The taxonomy is used by both the resume parser AND the JD parser:
- **Resume**: Discovers skills embedded in prose that NER and the skills section missed. College student resume went from ~20 to 92 detected skills.
- **JD**: Finds skills in unstructured job descriptions. Plain-text JDs went from 0 to 60+ extracted skills.
- **Archetypes**: Instant role-type matching — "you're 8% Backend Developer, 5% Full Stack" before you even paste a JD.

Resource file: `Resources/taxonomies/skill-taxonomy.txt` (role|skill format).

## Entity Lookup — 11K Companies, 7K Locations, 144 Industries

Known entities mined from LinkedIn (24K companies) and Adzuna (17K postings) via DuckDB:
- **11,578 companies** (2+ job postings each) — validates NER ORG candidates against real employers
- **7,756 locations** — confirms city/region extraction
- **11,505 job titles** — validates title extraction
- **144 industries** — from LinkedIn company classifications

When parsing a JD, entity lookup runs as the 5th signal layer. If NER says "Amazon" and entity lookup confirms it's a known company, confidence gets multi-source boosted via RRF. Adzuna company accuracy went from 10% to 100%.

## JD Parser Overhaul — 5-Layer RRF Fusion

Complete rewrite of job description extraction. The pipeline now has 5 parallel signal layers:

| Layer | Source | What it finds |
|-------|--------|---------------|
| 1 | **Structural** | Title/company from first line, explicit labels (`Company:`, `Location:`, `Salary:`), section headers, salary patterns (£/$€/RM/NNNk) |
| 2 | **NER** | ORG→company, LOC→location, Skills, JobTitle from both BERT models |
| 3 | **Taxonomy centroids** | 19,983 known skills via exact text matching |
| 4 | **LLM** | Full JSON extraction — the backstop that never fails. Expanded prompt: title, company, location, skills, education, seniority, industry |
| 5 | **Entity lookup** | Validates candidates against 11K+ known companies and 7K+ locations |

All signals fused via RRF with multi-source confidence boosting.

**Before → After (20 real-world JDs):**

| Field | Before | After |
|-------|--------|-------|
| Skills | 0 from plain-text JDs | 17–118 per JD |
| Company | 10% accuracy | 100% (Adzuna), 70%+ (plain text) |
| Salary | 0% on non-£ formats | 100% (£/$€/RM/NNNk) |
| Location | 50% | 85%+ |
| Education | 0% | ~60% |
| Preferred skills | 0 always | 3–16 per JD |
| Responsibilities | Never extracted | Now extracted from section headers |
| Benefits | Never extracted | Now extracted from section headers |

**New fields extracted:** education requirements, seniority level (from title), industry, hybrid flag, salary currency/period, responsibilities, benefits.

## Structural Extractor Improvements

- **Explicit label detection**: `Company:`, `Location:`, `Salary:`, `Type:` lines parsed with regex
- **Expanded salary patterns**: `$NNN,NNN`, `NNNk-NNNk`, `RM N,NNN – RM N,NNN per month`, single-value salaries
- **More section heading keywords**: 25 required-skill variants, 14 preferred-skill variants, responsibilities, benefits
- **Numbered list support**: `1. Item`, `2) Item` parsed alongside bullet points
- **Markdown artifact cleanup**: `**bold**` stripped from extracted text
- **Education detection**: Bachelor/Master/PhD/BSc/MSc patterns
- **Seniority detection**: Junior/Mid/Senior/Lead/Principal from title text
- **Hybrid detection**: "N days/week in office" patterns

## LLM Extraction Improvements

- **Expanded prompt**: Now requests 12 fields (was 7): added education, seniorityLevel, industry, isHybrid, responsibilities
- **Dedicated `ExtractJsonAsync`**: 4K response cap (was 2K via `ExtractSkillsAsync`)
- **Fixed chicken-and-egg bug**: LLM `IsAvailable` flag prevented first call. Removed gate — LLM always tries, fails gracefully.
- **Better JSON parsing**: Finds `{...}` boundaries in LLM response, handles preamble/postamble
