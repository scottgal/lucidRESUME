# v1.4.0 Release Notes

## Name Extraction: RRF Fusion + LLM Backstop

Complete rewrite of person name extraction. Previously used a fragile tiered fallback chain — now uses the same RRF fusion pattern as the rest of the extraction pipeline.

**Root cause fixed**: The resume NER model (`yashpwr/resume-ner-bert-v2`) was receiving **lowercased input** despite being a **cased model** (`bert-base-cased`). `OnnxNerOptions.LowerCase` defaulted to `true`. This single bug caused most name extraction failures.

**Architecture change**:
- **Before**: Sequential fallback — NER → heading → first line → give up
- **After**: All signals produce candidates simultaneously, fused with confidence boosting for multi-source agreement:
  - Signal 1: NER PersonName entities (both models)
  - Signal 2: Markdown headings (with OCR fix, concatenated name split, email cross-reference)
  - Signal 3: First meaningful lines (positional)
  - Fusion: group by normalised name, +0.10 confidence per agreeing source
  - Cross-reference: NER fragments expanded against longer positional matches ("Susan" → "Susan Forsythe")

**LLM backstop**: When fused confidence < 0.50 (single-word names, garbage NER output), the first 10 lines are sent to LLM for name extraction. Added `ExtractNameAsync` to all three providers (Ollama, Anthropic, OpenAI). Fast — only ~500 chars sent, sub-second response.

**Results across 26 test resumes**:
- Before: ~12/26 correct names (46%)
- After: 24/26 correct names (92%)
- Remaining 2: a template with placeholder names, and a Spanish resume with NER truncation

Specific fixes:
- "Led" → "Philipp Krenn" (NER false positive on action verb, now prefers multi-word name-like entities)
- "Full Time" → "Peter Müller" (concatenated PDF name + email cross-reference)
- "Susan" → "Susan Forsythe" (NER truncation, expanded via first-line cross-reference)
- "Bay" → "Monica Meyer" (LLM backstop catches single-word NER garbage)
- "E" → "Ernani Joppert P Martins" (LLM backstop)
- "Park" → "Claud D. Park" (LLM backstop)
- "F ELIPE M AGNO DE A LMEIDA" → "Felipe Magno De Almeida" (OCR spacing fix)

## Batch Testing & Quality Evaluation

- `batch-test` command parses all resumes in a directory, outputs per-file JSON + summary CSV
- `evidence` command with `--output` exports full skill ledger as structured JSON
- Evaluated all 26 test resumes (7 languages, multiple formats) — identified and fixed parser quality issues

## Bug Fixes

- **NER model casing**: Resume NER model was receiving lowercased input for a cased vocabulary — fixed by setting `LowerCase = false`
- **Non-name phrase rejection**: "Full Time", "Part Time", "Work History", "Professional Experience" etc. no longer extracted as person names
- **Heading phone stripping**: "Luiza Maria CICONE H 07 81 36 81 44" → strips trailing phone numbers
- **OCR spacing repair**: "F ELIPE M AGNO DE A LMEIDA" → merges single-letter OCR fragments
- **Concatenated PDF names**: "petermüller" → "Peter Müller" using email cross-reference
- **Document title skipping**: Headings like "RESUME", "CV", "CURRICULUM VITAE" no longer treated as names
- **ALL CAPS normalisation**: "MONICA MEYER" → "Monica Meyer" for both heading and LLM results
